## Design Context

### Users
Alternative finance professionals: fund managers, underwriters, operations leaders at firms that provide small business financing (MCA, revenue-based lending, factoring). They evaluate hundreds of deals daily, are time-constrained, and need tools they can trust with high-stakes credit decisions. They arrive at undersight.ai to understand whether the platform can automate and improve their existing workflow. Secondary audience: developers evaluating API integration (Docs section).

### Brand Personality
- **Voice:** Professional yet approachable, confident, tech-forward
- **3-word personality:** Intelligent. Precise. Trustworthy.
- **Emotional goals:** Confidence that this technology works, clarity about what it does, trust that it handles sensitive financial data responsibly
- **Persona:** "The insightful analyst who sees what others miss"
- **Name rules:** "undersight" always lowercase. "underchat" always lowercase. Never capitalize.

### Aesthetic Direction
- **Visual tone:** Minimal, clean, modern. Editorial quality with monochrome illustration style. Light-first with dark mode support.
- **References:** Equabli (content quality, polish, B2B SaaS credibility), PaveFi (domain communication clarity), Truffles (structural organization)
- **Anti-references:** No gradients, no playful/bubbly aesthetics, no stock photography, no aggressive color blocks. The mixed dark/light/amber section bands tested poorly — keep page rhythm uniform and light with subtle accents only.
- **Illustration style:** Monochrome editorial (matching existing blog assets: simple line art, grayscale, conceptual). All placeholder graphics use this style.
- **Theme:** Light mode default. Dark mode via `prefers-color-scheme: dark`.

### Copy Rules
- Never use "merchant" — say "business owner", "applicant", or "client"
- Never use "SMB" — say "small business"
- No em-dashes
- "undersight" and "underchat" always lowercase

### Design Principles

1. **Clarity over cleverness.** Every element must communicate its purpose immediately. Fund managers scanning the site should understand what undersight does within 5 seconds. No ambiguity, no jargon walls.

2. **Restraint is the accent.** Amber Rust appears only on interactive elements (buttons, links, active states). Its impact comes from scarcity. When everything is highlighted, nothing is.

3. **Trust through precision.** Typography, spacing, and alignment must be meticulous — 8px grid, consistent padding, token-driven values only. Sloppy details erode trust for a product that handles financial decisions.

4. **Content from the source.** 100% of text content comes from Fibery CMS. The site is a rendering layer, not a content store. This keeps copy editable without code changes.

5. **Uniform light, subtle depth.** Page rhythm stays light and clean. Visual hierarchy comes from typography weight, spacing, and the occasional Amber Rust accent — not from alternating dark/colored section backgrounds.

### Accessibility
- WCAG AA minimum (AAA where achievable)
- Amber Rust on Graphite 900: 4.2:1 (AA for large text only — use only on buttons with white text or large/bold text)
- Paper White on Graphite 900: 13.5:1 (exceeds AAA)
- Keyboard navigation: all interactive elements focusable with visible focus rings
- `prefers-reduced-motion` respected for all animations
- Semantic HTML throughout
